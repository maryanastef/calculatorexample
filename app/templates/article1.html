AAA Testing with Python

What is AAA Testing?

AAA Testing, also known as Arrange-Act-Assert, is a great way to structure test cases. It describes the order of operations that need to be performed to write a good test. AAA Testing represents a recognizable test template following the A-A-A pattern of testing. The pattern focuses each test on a single action.

By applying AAA, you can ensure that your test is clear and more understandable and manageable.

For more information on AAA Testing, please read the article Arrange-Act-Assert

The A-A-A Pattern (Arrange-Act-Assert):
  1. Arrange: inputs and targets. Arrange steps set up the test case. It will tell you if the test requires any objects or special settings, if it needs to log into a web app, etc.
  2. Act: acts on the target behavior. Act steps cover the main thing to be tested, like calling a function or method.
  3. Assert: asserts expected outcomes. Act steps return some sort of a response and Assert steps verify the “goodness or badness” of that response. This step determines if the test PASSES or FAILS. Sometimes, assertions can be simple, like checking if a calculator adds, subtracts, multiplies or divides numbers properly. Other times, assertions can require checking multiple facets of a system.

Unit Testing:

Unit Testing is a method that is used to test programs and applications to see if they function correctly and without any problems. It is part of software testing, which involves many steps. If any problems occur, they should be taken care of early on, so that they don't show up later for the end users.

The main purpose behind unit testing is to check that all individual parts are working well like it is supposed to. The entire system will only be able to work well if the individual parts are working well.

There are two types of unit testing: manual and automated. The automated method is most preferred because it is faster and more accurate. Automated testing usually involves the developer first writing a section of the code in the application so the function can be tested. The manual approach has a step-by-step instructional procedure that helps developers perform the test.

Pytest:

Pytest is another framework for testing that allows developers to write test codes using Python. An advantage of a pytest is that it can run tests in parallel, it can run specific tests, or it can skip tests that you want it to skip.

In order to use Pytest, you need to install it:

pip install pytest
