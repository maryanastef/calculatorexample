Implementing Pylint & Glossary

Pylint Manual

Pylint is a tool that checks for errors in your Python code. It analyzes your code and compares it to the coding standard. If you are coding in Python, the computer does not catch bugs for you. Therefore, you need a lint tool, like Pylint, to find errors.

The default coding style used by Pylint is similar to PEP 8.

Installation:

Assuming that you are using Python, Pylint should be easily installable using pip. Install Pylint by running this command:

pip install pylint

Check where Pylint was installed by running:

$ which pylint
/usr/local/bin/pylint

For more information, please visit the Installation Guide

Running Pylint:

Pylint is meant to be called from the command line. You can type:

pylint fileName.py

Or you can call the directory, followed by the file. This will only work if the directory is a python package and has an __init__.py file.

pylint directory/fileName.py

For more information, please visit the Running Pylint Guide

Command Line Options:

Pylint has many checkers. You can disable a specific checker and its messages by specifying:

--disable= <symbol>

If you want to enable only some checkers or message symbols, first use

--disable=all

Then,

--enable=< symbol>

where, < symbol > is in a comma-separated list of checker names and message symbols.

It is possible to disable all messages with --disable=all. This is useful to enable only a few checkers or few messages by first disabling everything and then re-enabling what you need.

Then, Python will search for a configuration file named pylintrc or .pylintrc

Open in PyCharm:

GLOSSARY

These are useful terms to know for object-oriented programming

Attribute: creates attributes on class instances inside methods.

Class Method: means that it is bound to the class and not the object of the class. This is useful for when we need to access the class itself and not the object. Class method can change a class state that would apply across all the instances of a class.

Class: is a code template for creating objects. In Python, a class is created by using the keyword "class".

Constructor: is a special kind of method that Python calls when it instantiates an object using the definitions found in your class. Python relies on the constructor to perform tasks such as assigning values to variables that the object will use.

Design Patterns: provide a repeatable solution to a commonly occurring problem in software design. It is an essential part of object-oriented software development.

Façade: is a structural design pattern that provides a simplified interface to a complex system of classes. It is often used in object-oriented programming.

Factory: is an object for creating other objects

Fixture: defines the steps and data that constitutes the arrangement phase of a test. In pytest, fixtures are functions used to define a test’s act phase.

Instantiation: is creating a copy of the class which inherits all class variables and methods. To instantiate a class, we have to call the class as if it were a function, passing arguments that the __init__ method defines.

Method: is a function that “belongs” to an object. For example, list objects have methods called append, insert, remove, sort , etc. In Python, you can create a list , list =[10,20,30] and apply a method such as my_list.append(40). This will add the number 40 to the existing list.

Namespace: is a dictionary in which the keys are the object names and the values are the objects themselves. Python maintains a namespace in the form of a Python dictionary.

Object: is the collection of various data and functions. Everything in Python is treated as an object (variables, functions, lists, tuples, dictionaries, etc). It is important to note that every object belongs to its class.

Property: is a function in Python that returns the object of the property class and is used to create a property of the class.

Static Method: means that it belongs to the class rather than the class’s object. A static method can be called without creating an object for the class. We can directly call the static method by referencing the class name.

Type Cast: is a method to convert one variable data type into another data type. This can be useful when you want to add two numbers in which one existing value of a variable is an integer and the second is a string.

Type Hint: is a formal solution to indicate the type of value within your Python code. Type hints help you maintain cleaner code and improve linters.

Unit Test : is the first level of software testing. This guarantees that some change in the code won’t affect the code that is running without errors.
